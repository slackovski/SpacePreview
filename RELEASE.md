# SpacePreview 1.1.0

## 🎉 First Stable Release

A lightweight macOS Quick Look extension for syntax highlighting code and documents directly in Finder with a single Space key press.

## ✨ Features

- **Multi-Language Support**: Syntax highlighting for 63+ programming languages
- **Quick Preview**: Press Space in Finder to instantly preview any supported file
- **Smart Theme Detection**: Automatically matches macOS light/dark mode
- **Markdown Rendering**: Renders `.md` files as formatted HTML
- **Line Numbers**: Code is displayed with line numbers for easy reference
- **Beautiful Highlighting**: Uses Atom One color scheme for syntax highlighting
- **Fast & Lightweight**: No network access, self-contained bundle
- **Universal Binary**: Supports both ARM64 (Apple Silicon) and Intel architectures
- **Modern Architecture**: Built as App Extension, fully compatible with macOS 15 Sequoia

## 🎯 Supported File Types (63+ formats)

### Languages
JavaScript, TypeScript, Go, Java, Kotlin, Rust, PHP, Python, Ruby, Shell, Swift, Objective-C, C, C++, Vue, CSS, SCSS, SASS, HTML, XML, JSON, YAML, TOML, SQL, GraphQL, Protobuf, Terraform, Makefile, Markdown, Lua, Dart, Scala, Elixir, LaTeX, reStructuredText, Subtitles (SRT, ASS, SSA), and more!

## 📥 Installation

1. **Download** SpacePreview.dmg from the releases below
2. **Mount** the DMG file by double-clicking
3. **Drag** SpacePreview.app to your Applications folder
4. **Open** the app once (the extension will be automatically enabled)
5. **Enjoy** — Press Space in Finder to preview your code!

## ⚙️ System Requirements

- macOS 13 Ventura or later (including macOS 15 Sequoia)
- Apple Silicon (ARM64) or Intel processor
- ~15 MB disk space

## 🐛 Known Limitations

The following file types are **not currently supported**:
- `.ts` (plain TypeScript) — use `.tsx` or `.cts` instead
- `.jsx` (JSX) — use plain `.js` or `.tsx` instead
- `.mts` (TypeScript modules) — use `.tsx` or `.cts` instead
- `.cs` (C#)
- `.vtt` (WebVTT subtitles) — use `.srt` instead
- `.ps1` (PowerShell)
- `.fish` (Fish shell) — use `.sh` or `.bash` instead
